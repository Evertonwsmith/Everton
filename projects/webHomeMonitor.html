V1.0 Project Overview

The goal of this project was to first create a Home Assistant interface to sync and control all of my smart home devices. I then wanted to create a Home Assistant controlled robotic unit that would let me monitor my home via camera feed, temperature sensing, and humidity sensing. Other features may be added and documented as the project continues.

V1.0 System Architecture

The robot utilizes a "Primary-Secondary" communication model:

  • Node A (Vision): ESP32-S3 WROOM-1 (Octal PSRAM). Communicates via WiFi/API to Home Assistant.
  • Node B (Control): ESP32 + Arduino. The ESP32 handles the HA API and high-level logic; the Arduino handles PWM for motors and timing-sensitive sensor readings (DHT11).

Integration: Home Assistant acts as the Command & Control center, hosting the MJPEG stream and control entities.

Revisions

After initial testing and research, this project is being pivoted to a more efficient and secure networking structure.

The single ESP32-S3 will handle both vision and control functions, reducing complexity and improving performance.

This will be connected to a local network via HTTP, and WebSockets will be used to facilitate real-time communication.

4. Node B: Control System (ESP32 + Arduino + Motors)

4.1 Motor Control (L298N / Dual H-Bridge)

Driving four TT motors requires two H-Bridge drivers (or one if wired in parallel per side).

Function Arduino Pin Description
ENA D9 (PWM) Speed Control (Left Side)
IN1 / IN2 D8, D7 Direction (Left Side)
IN3 / IN4 D11, D12 Direction (Right Side)
ENB D10 (PWM) Speed Control (Right Side)

4.2 Environmental Sensor (DHT11)

  • Pin: Arduino D5 (Digital)
  • VCC: 5V (or 3.3V)
  • GND: Common Ground

5. Software Integration (ESPHome)

Motor Control Logic

On the Control ESP32, use the template platform to create buttons or a vacuum-style remote component in Home Assistant. This sends signals to the Arduino via UART or I2C.

Sensor Logic (DHT11)

The DHT11 is read by the Arduino and passed to the ESP32.

6. Home Assistant Dashboard Setup

To unify the build, create a Picture Elements Card or a Grid Card:

  • Background: The camera.robocam entity set to "Live".
  • Overlays: Buttons: Up, Down, Left, Right arrows mapped to the Control ESP32 scripts.
  • Badges: Current temperature and humidity readings from the DHT11 sensors.
